134  size_t pushBack(const char* ptr, size_t sz, bool keepLastOnFlash = true)
135  {

278  // This is a placeholder for the function responsible to split large payload to pieces
279  // fitting 8kB CRU pages.
280  // The RawFileWriter receives from the encoder the payload to format according to the CRU format
281  // In case this payload size does not fit into the CRU page (this may happen even if it is
282  // less than 8KB, since it might be added to already partially populated CRU page of the HBF)
283  // it will write on the page only part of the payload and carry over the rest on extra page(s).
284  // By default the RawFileWriter will simply chunk payload as is considers necessary, but some
285  // detectors want their CRU pages to be self-consistent and in case of payload splitting they
286  // add in the end of page to be closed and beginning of the new page to be opened
287  // (right after the RDH) detector-specific trailer and header respectively.
288  //
289  // The role of this method is to suggest to writer how to split the payload:
290  // If this method was set to the RawFileWriter using
291  // RawFileWriter::setCarryOverCallBack(pointer_on_the_converter_class);
292  // then the RawFileWriter will call it before splitting.
293  //
294  // It provides to the carryOverMethod method the following info:
295  // rdh : RDH of the CRU page being written
296  // data : original payload received by the RawFileWriter
297  // ptr : pointer on the data in the payload which was not yet added to the link CRU pages
298  // maxSize : maximum size (multiple of 16 bytes) of the bloc starting at ptr which it can
299  // accomodate at the current CRU page (i.e. what it would write by default)
300  // splitID : number of times this payload was already split, 0 at 1st call
301  // trailer : what it wants to add in the end of the CRU page where the data starting from ptr
302  // will be added. The trailer is supplied as an empy vector, which carryOverMethod
303  // may populate, but its size must be multiple of 16 bytes.
304  // header : what it wants to add right after the RDH of the new CRU page before the rest of
305  // the payload (starting at ptr+actualSize) will be written
306  //
307  // The method mast return actual size of the bloc which can be written (<=maxSize).
308  // If this method populates the trailer, it must ensure that it returns the actual size such that
309  // actualSize + trailer.size() <= maxSize
310  // In case returned actualSize == 0, current CRU page will be closed w/o adding anything, and new
311  // query of this method will be done on the new CRU page
312 
313  int carryOverMethod(const RDHAny*, const gsl::span<char> data, const char* ptr, int maxSize, int splitID,
314  std::vector<char>& trailer, std::vector<char>& header) const
315  {
316  return maxSize; // do nothing
317  }
